# Paging Quality Checklist

- Does this page correspond to meaningful user harm?
- Can the responder tell what first action to take?
- Is there a runbook link and clear owner?
- Is this truly page-worthy, or should it be a ticket?
- Is the policy tested against historical noise?
- Is there dedupe or suppression for obvious shared-cause incidents?
- Does the alert distinguish telemetry absence from service failure?
- Would repeated firing of this page improve response, or only create fatigue?
