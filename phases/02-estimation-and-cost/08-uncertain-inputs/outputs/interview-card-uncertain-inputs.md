# Interview Card — Uncertain Inputs

## Default script

- "I don’t have exact inputs, so I’ll use a low, base, and high case."
- "I’ll optimize the architecture for the base case and name the threshold where it changes."
- "The most important uncertainty here is ____ because it moves ____."

## Common misses

- waiting for perfect data
- hiding uncertainty
- varying too many inputs at once
- claiming the same architecture works at any scale
