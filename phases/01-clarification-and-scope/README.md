# Phase 01 — Clarification & Scope Control

~8 hours.

This phase teaches the senior-level behavior that changes every later design decision: clarifying the prompt, narrowing scope honestly, making assumptions explicit, and agreeing on priorities before drawing architecture.

Lessons:

- `01-functional-vs-non-functional` — separate user-visible capability from system-quality constraints
- `02-high-leverage-questions` — ask the few questions that actually change storage, routing, consistency, and cost
- `03-scope-cuts` — trim the problem without dodging the intent
- `04-assumption-logging` — keep momentum when the interviewer leaves ambiguity unresolved
- `05-prioritization` — negotiate which requirements dominate the design
- `06-workload-shape` — translate user journeys into read/write, fanout, and burst patterns
- `07-anti-patterns` — learn the common senior-level failure modes in clarification
- `08-prompt-reframing` — restate an ambiguous prompt into a designable v1 with explicit trade-offs

By the end of the phase, the learner should be able to reach a crisp scope and directional sizing story within the first few minutes of an interview.
